Displaying a Static Web Page with Django

Introduction

This page describes the process of displaying a static web page using Django, starting from the request and ending with the response. The example demonstrates how a simple template is rendered by a view in Django.

Steps in the Process

  1. URL Configuration: The process begins with defining a URL pattern that maps a URL to a specific view. This is done in the urls.py file.
  2. View Function: The URL pattern points to a view function in the views.py file. This view function handles the request and returns a response.
  3. Template Rendering: The view function uses a template to generate the HTML content. The template is a simple HTML file stored in the templates directory.
  4. HTTP Response: The rendered template is returned as an HTTP response to the client's browser, which then displays the static web page.

Example Code

1. URL Configuration

In the urls.py file:


from django.urls import path
from . import views

urlpatterns = [
path('static-page/', views.static_page, name='static_page'),
]

2. View Function

In the views.py file:


from django.shortcuts import render

def static_page(request):
return render(request, 'static_page.html')

3. Template Rendering

In the static_page.html file, located in the templates directory:

Welcome to the Static Page

Explanation

When a user navigates to /static-page/, the following sequence of events occurs:

  1. The URL pattern defined in urls.py matches the request URL and calls the static_page view function.
  2. The static_page view function in views.py handles the request and uses the render function to generate the HTML content from the static_page.html template.
  3. The rendered HTML content is returned as an HTTP response to the client's browser.
  4. The client's browser displays the static web page with the content defined in the template.

Conclusion

This simple example demonstrates how Django handles a request to display a static web page using a view and a template. The process involves URL configuration, view function handling, template rendering, and returning an HTTP response.
